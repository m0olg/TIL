# 오류처리
스위프트에서 오류는 Error라는 포로토콜을 준수하는 타입의 값을 통해 표현됨 \
Error 프로토콜은 사실상 요구사항이 없는 빈 프로토콜일 뿐이지만, 오류를 표현하기 위한 타입(주로 열거형)은 이 프로토콜을 채택함!
## 01 오류표현, 오류 던지기
Error 프로토콜과 열거형을 통해서 오류를 표현

Swift 열거형은 관련된 오류 조건의 그룹으 모델링 하는데 특히 적합, \
연관값을 사용하여 오류의 특성의 추가 정보를 전달 가능

```swift
// 정의
enum HachiwareError: Error { // 하치와레 오류 종류를 만들어보겟다~
    case tooSpicy(level: Int) // 너무너무너뮤매움ㅜㅜ(맵기숫자같이저장)
}

// 함수
func eat() throws { // throws는 오류를 낼 수도 있어라는 표시
    throw HachiwareError.tooSpicy(level: 5) // 실제로 오류를 발생시켜보겟따 (맵기 : 5)
    // throws니까 이게 오류 던지기임!!!!!!!!!!!!⭐⭐⭐
}

// 실행
do { // 오류 처리 시작
    try eat() // 오류 날 수도 있는 함수 실행
} catch HachiwareError.tooSpicy(let level) { // 오류 잡기 (let level : 전달 받은 맵기 값 꺼내기)
    print("하치와레 : 너무 매워ㅠㅠㅠㅠㅠㅠㅠㅠㅠㅠㅠㅠ \(level)") // 오류가 발생하게 되면 실행되는 코드
}
```
**= 만들고 (enum) -> 던지고 (throw) -> 잡는다 (catch)**

###### 이처럼 오류 발생의 여지가 있는 메서드는 throws를 사용, 오류를 던질수도 있지만 이에 대비하여 던져진 오류를 처리하기 위한 코드도 작성해야함 예를 들어 던져진 오류가 무엇인지 판단하여 다시 문제를 해결한다든지, 다른 방법으로 시도해본다든지 사용자에게 오류를 알리고 사용자에게 선택 권한을 넘겨주어 다음에 어떤 동작을 하게 할 것인지 결정하도록 유도하는 등의 코드를 작성해야 함!

## 02 오류 타입 지정
오류는 `Error` 프로토콜을 따르는 값
어떤 오류가 날지 미리 다 알 수 없음
-> 그래서 보통 타입 제한 없이 위에서 말한 오류 던지기 (`throws`)를 사용
### 기본 방식
```swift
func test() throws()
```
- 모든 종류의 오류 가능 (`any Error`)
```swift
func test() throws(MyError) { }
```
- 이 함수는 특정 오류만 던짐!!
- 그래서 다른 오류를 던지면 컴파일 에러가 일어남
