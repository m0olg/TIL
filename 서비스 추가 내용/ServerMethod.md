# 01 Server Method이란?
→ 클라이언트가 서버에 어떤 요청을 보낼지 나타내는 방식

<br>
<Br>

# 02 종류
1. **GET** : 서버의 **데이터를 조회**할 때 사용
    - 특징 : 데이터를 가져오기만 하므로 서버의 상태를 변경하지 않는 안전한 메서드
    - 데이터 전송 : 보낼 데이터가 있다면 URL 뒤에 주렁주렁 붙이는 **Query String**(쿼리 스트링) 방식 사용 (ex. `?id=12`)

2. **POST** : 서버에 **새로운 데이터를 생성**할 때 사용
    - 특징 : 같은 요청을 여러 번 보내면 서버에 똑같은 데이터가 여러개 생성
    - 데이터 전송 : 데이터를 외부에서 볼 수 없게 편지 봉투 안인 HTTP Body에 숨겨서 보냄

3. **PUT** : 기존 **데이터를 전체 수정**할 때 사용
    - 특징 : 수정할 데이터를 통째로 덮어쓰는 방식 \
    만약 보내지 않은 필드가 있다면 그 필드는 비어버리거나 초기화될 수 있음
    - 멱등성 : 같은 요청을 10번 보내든 100번 보내든 서버의 최종 결과는 항상 똑같음 (이를 멱등성이 있다고 함)

4. **PATCH** : 기존 **데이터의 일부를 수정**할 때 사용
    - 특징 : PUT과 달리 바꾸고 싶은 특정 필드 (ex. 이름만 변경)의 데이터만 쏙 골라서 수정

5. **DELETE** : 서버의 **데이터를 삭제**할 때 사용
    - 멱등성 : 특정 리소스를 지정해서 삭제, `PUT`과 마찬가지로 여러 번 실행해도 결과가 똑같으므로 멱등성을 가짐

<br>
<br>

+ 나중에 서버도 클라이언트에게 요청을 하는데 그땐 HTTP 방식이 아니고 Socket이라는 방식 사용 (알고만 있기)

+ HTTP는 클라이언트가 먼저 말을 걸어야만 서버가 대답하는 '단반향 통신'이지만 \
Socket(웹소켓)은 실시간 카카오톡 채팅처럼 클라이언트가 서로 언제든 말을 걸 수 있는 **실시간 양방향 통신**
