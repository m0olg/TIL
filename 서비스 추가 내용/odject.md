# 01 Object(객체)란?
→ 데이터와 기능을 함께 가지고 있는 것

```swift
class Cat {
    // 데이터 ( 상태 / 프로퍼티 )
    var name: String = "나비"
    var age: Int = 3

    // 기능 ( 행동 / 메서드 )
    func meow() {
        print("\(name)가 쥐랑 술래잡기를 합니다")
    }
}
```

# 02 객체와 Class의 차이
## 2-1 Class
객체를 만들기 위한 공장 = 설계도
```swift
class Dog {
    var name = "멍멍이"
}
```
- `class Dog { }`는 실제로 `name`이 `멍멍이`인 Dog 객체가 생성된 것이 아니고 Dog 객체를 생성하면 기본적으로 `멍멍이`라는 값을 가지도록 정의한 설계도임
- 그냥 '틀'만 있는 상태
- 클래스만 있으면 실제 데이터는 없음
- 클래스는 코드로 작성된 타입 정의일 뿐, 그 자체로는 실행 가능한 공간이 없음
- 설계도면 상태이기 때문에 스마트폰의 메모리를 전혀 차지하지 않음

## 2-2 Object
클래스를 통해 실제로 만들어져 사용할 수 있는 데이터
```swift
let dog = Dog()
```
- `let dog = Dog()`는 Dog 클래스라는 설계도를 통해 실제로 생성된 객체
- 이 객체는 실제 메모리에 생성되며 '멍멍이'라는 `name`값을 가지게 됨
- 메모리 할당 : `Dog()`를 호출하여 객체를 만드는 순간 스마트폰의 메모리 영역에 실제 데이터가 올라감
- 참조 타입 : `dog` 변수 안에는 실제 데이터가 아닌 메모리에 있는 데이터의 '주소(위치)'가 저장됨

### 정리

|  | Class (클래스) | Object (객체) / Instance |
| :--- | :---: | :---: |
| **비유** | 건축 설계도, 붕어빵 틀 | 실제 완공된 집, 실제 붕어빵 |
| **실체** | 개념적인 정의 (코드) | 메모리에 존재함 (실제 데이터) |
| **메모리(RAM)** | 차지하지 않음 | (Heap) 영역의 메모리를 차지함 |
| **수량** | 단 1개만 정의됨 | 설계도 하나로 무한정 생성 가능 |
| **코드 예시** | `class Dog { ... }` | `let myDog = Dog()` |
