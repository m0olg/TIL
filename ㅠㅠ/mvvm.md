# 01 mvvm의 개념
#### (1) 아키텍처 패턴이란?
영어 단어를 그대로 해석하면 건축학, 설계학을 뜻함 \
개발에서 의미하는 건 **개발 프로그램의 구조와 뼈대**를 의미 \
IOS에서 가장 기본이 되는 아키텍처 패턴은 mvc와 mvvm이 있음


(그림 이따가 기숙사 가서 넣기 ⭐️⭐️⭐️)

== mvvm은 UI와 Logic을 분리해서 유지보수와 재사용, 코드가 날아갈 위험을 줄이는 아키텍처의 패턴 중 하나 !! \
이제 하나씩 알아보자
### 1-1 View (UI)
이건 쉽게 말해서 그냥 보여주기식임 사용자가 보는 화면 그 자체로 버튼이나 라벨, 이미지, 텍스트 필드 등 \
눈에 보이는 모든 것을 말함
### 1-2 Controll (VC)
View와 Logic을 연결하는 징검다리 역할을 하는 파일임 \
하나의 파일이 책임이 올라가는 걸 줄이기 위해 분리를 시키고 이제 그걸 이어줘야 하는데 그 역할을 이 파일이 함

이 두 개를 연결해야지 정상 작동하기 때문에 사용자에게 View나 Logic 파일을 보여줄 수도 있지만 정상 작동을 안 하기 때문에 Controll 파일을 보여준다 (**사용자가 보는 파일임 👀**)
### 1-3 Logic (VM)
로직은 로그인을 판단하고 데이터 가공이나 서버 요청 등의 역할을 함 (변화 시켜주는 기능) \
이건 백엔드 개발자 관련 부분이라 보통 자바스크립트로 많이 코드를 쓰는데 자바를 Xcode는 이해 못하기 (서버고유언어로 줘서) 때문에 우리가 이진수를 몰라서 진법 변환을 하는 것처럼 계산을 통해 Xcode가 알아 먹을 수 있게함 !!
### 1-4 model
Logic ViewModel을 구성하는 작은 계산을 해주는 파일임 \
모델이랑 합쳐서 ViewModel을 만듦
# 02 UI와 Logic을 같은 폴더 안에 넣을 수 있을까?
➜ 결론부터 말하면 가능하다
- 같이 넣은 파일을 mvc라고 하고
- 다른 폴더로 분리 시킨 걸 오늘 배운 mvvm이라고 함
하지만 같은 곳에 넣게 되면 그 파일에 대한 **책임이 올라가고** 그 파일이 날아가면 고생하게 됨 😿

# 03 호출
- `UI()`, `Logic()` → 호출 (파이썬에서 배운 함수 호출이랑 거의 똑같다고 보면 됨) \
사용자가 보는 파일은 Controll 파일이므로 호출을 해야함

UI와 Logic 두 개를 다른 파일로 분리해서 UI가 Logic의 기능을 쓰고 싶을 때 호출 해야함 ‼️
