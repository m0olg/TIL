# mvvm
아키텍처 패턴 중 하나의 종류 \
UI   <-   Controll <- Logic <- model \
보여주기식      파일      기능

= 서버에서 로그인에 대한 기능을 swift언어로 바꾸어야함 (변화시켜주는 기능이 로직임)

- UI랑 Logic을 UI라는 폴더 안에 같이 넣을 수 있음 mvc \
다른 폴더로 분리 시킨게 mvvm \
mvc로 하게 되면 그 파일에 대한 **책임이 올라가고** 그 파일이 날아가면 끝나버림

Logic -> 서버에서 로그인 기능을 줬는데 이건 서버만 알아먹는 언어로 줬기 때문에 (서버고유언어) Xcode가 알아먹을 수 있도록 변환 시켜줌 \
우리가 10진법으로 바꿀 때 계산이 들어가는 것처럼 똑같이 계산을 통해서 Xcode가 알아 먹을 수 있게함

그래서 책임을 줄이기 위해 분리를 시켜두면 이제 그걸 이어줘야하는데 Controll이 징검다리 역할을 함

`UI()`, `Logic()` -> 호출 \
사용자가 보는 파일은 Controll 파일 (UI와 Logic 두 개를 이어준 파일이기 때문에)

- **UI** : view
- **Controll** : VC
- **Logic** : VM

- 모델은 로직이라는 뷰 모델을 구성하는 작은 계산을 해주는 파일

모델이랑 합쳐서 뷰모델을 만ㄷ름ㅇ
